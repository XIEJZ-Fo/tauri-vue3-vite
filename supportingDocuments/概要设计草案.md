### **todoList 个人学习辅助工具概要设计草案**

#### **1. 系统架构与文件结构设计**

**物理架构**：单节点部署的桌面应用程序架构，由 Tauri 框架打包，但是完全由vue3开发功能。

**逻辑架构**：扁平化模块结构。整个应用由 `app.vue` 作为根组件，协调并展示三个核心功能模块和一个数据存储模块。模块间通过定义清晰、数量有限的接口进行通信，以实现松耦合。

**项目文件结构**：
```
/src
├── app.vue                    # 应用根组件，负责布局和模块间高层协调，引用其他.vue文件的<template>进行统合展示
├── /modules
│   ├── todolist.vue           # 任务管理模块：包含任务数据模型、管理逻辑及UI
│   ├── pomodoro.vue           # 番茄钟模块：包含计时引擎、状态机及UI
│   ├── stats.vue              # 数据统计与展示模块：包含计算逻辑及图表UI
│   └── storage.ts             # 数据持久化模块：封装所有本地JSON文件的读写操作
```

#### **2. 核心模块设计**

**2.1 任务管理模块 (`/src/modules/todolist.vue`)**
*   **职责**：完整实现 **FR-001** 的所有功能。
*   **内部构成**：
    *   **数据模型**：定义 `Task`、`Category` 等核心类。`Task` 包含名称、描述、优先级、创建截止日期、完成的番茄钟数量、状态等属性。使用组合模式管理任务与子任务的树形结构。`Category`即父类标签，决定当前待办事项所属。展现形式为默认展示父类待办事项列表，点击父类展开子类待办事项。父类事项创建时用户需要输入名称、描述、优先级、创建截止日期，完成的番茄数则统计子事项的总和，子事项开启番茄钟时将状态与其同步。
    *   **业务逻辑**：直接在 Vue 3 的 `setup` 或 `<script setup>` 中实现任务的增删改查（CRUD）、状态更新、分类管理等所有操作函数。
    *   **用户界面**：在 `<template>` 部分实现任务列表的展示、编辑表单、操作按钮等全部UI。
*   **关键设计**：
    *   模块内部维护当前的任务列表状态（使用 Vue 的 `reactive` 或 `ref`）。
    *   对外提供一个简化的服务接口（外观），例如一个 `taskService` 对象，包含 `getActiveTask()`、`setTaskActive(taskId)` 等方法，供其他模块（如番茄钟模块）调用。

**2.2 番茄钟模块 (`/src/modules/pomodoro.vue`)**
*   **职责**：完整实现 **FR-002** 的核心逻辑与展示。
*   **内部构成**：
    *   **引擎类 (`PomodoroEngine`)**：在 `<script>` 中定义，实现一个状态机，管理“工作中”、“休息中”等状态及其转换。负责计时、暂停/继续一个周期等核心逻辑。注意，一个番茄钟是一个work+一个break，而停止则是结束番茄钟周期计时的按钮，同时数据的保存和本地写入都是以一个番茄钟为周期，而不是以开启番茄钟和停止番茄钟为周期。
    *   **业务逻辑**：处理计时事件，并在完成一个番茄钟时，通过调用 `taskService` 接口更新对应任务的已完成番茄钟数量（**FR-002.3**）。
    *   **用户界面**：展示当前计时器状态、剩余时间、关联的任务信息（**FR-003.2**），并提供开始、暂停、重置等控制按钮。
*   **关键设计**：
    *   通过注入（`provide`/`inject`）或 `props` 接收来自 `todolist.vue` 的 `taskService` 接口，以获取和设置当前活动任务。
    *   计时结束时，若关联了任务，则通过 `taskService` 通知任务模块更新进度。

**2.3 数据统计与展示模块 (`/src/modules/stats.vue`)**
*   **职责**：完整实现 **FR-003** 的数据计算与可视化。
*   **内部构成**：
    *   **数据逻辑**：从 `storage.ts` 模块读取原始数据，在组件内部计算“今日统计”（完成番茄数、完成任务数）和“学习进度总览”（按日/周的分布图）。
    *   **用户界面**：使用图表库（如ECharts或Chart.js）或纯CSS/HTML展示统计摘要和趋势图表。
*   **关键设计**：
    *   为提高性能（**NF-001.1**），对计算量较大的历史统计数据可进行缓存。
    *   通过 `storage.ts` 模块提供的统一接口异步获取数据。

**2.4 数据持久化模块 (`/src/modules/storage.ts`)**
*   **职责**：确保 **NF-002.2** (数据持久性) 和 **NF-003.1** (容错性) 的关键模块。为上层所有 `.vue` 模块提供统一、安全的数据存取服务。
*   **导出接口**：提供 `loadData()`、`saveTasks(taskList)`、`savePomodoroRecord(record)` 等纯函数或异步函数。
*   **关键设计**：
    *   **原子化写操作**：采用“写时复制”策略。先将数据序列化到临时文件，确保写入完成后，再原子性地重命名覆盖原文件，防止数据损坏。
    *   **自动保存**：业务模块（如`todolist.vue`）在数据变更后调用保存接口，实现接近实时的自动保存。
    *   **崩溃恢复**：应用启动时，`storage.ts` 的 `loadData()` 函数会检查数据一致性。例如，如果发现存在状态为“进行中”但无对应活动计时器的任务，则将其状态重置为“未开始”。
    *   **数据存储**：所有数据明文存储于用户个人目录下的单一JSON文件中，符合**NF-004.1**的简化加密安全约束。

**2.5 应用根组件 (`/src/app.vue`)**
*   **职责**：应用的入口和布局框架。负责导入并组织 `todolist.vue`、`pomodoro.vue`、`stats.vue` 三个核心功能模块的显示。
*   **设计**：采用简单的布局（如上下或左右分栏），将各功能模块作为子组件放置。在此处完成各模块间必要服务接口的**提供**（`provide`），例如将 `todolist.vue` 导出的 `taskService` 提供给 `pomodoro.vue` 使用。

#### **3. 模块间通信与关键设计决策**

*   **外观模式 (Facade Pattern) 应用**：
    *   `todolist.vue` 对外暴露一个精简的 `taskService` 对象（如 `{ getActiveTask, updateTaskPomodoroCount }`）。
    *   `pomodoro.vue` 通过 Vue 的依赖注入机制获取该 `taskService` 并使用，而非直接操作任务数据。这最小化了模块间的耦合。
*   **数据流**：
    1.  用户操作触发 `todolist.vue` 或 `pomodoro.vue` 内部的函数。
    2.  函数更新模块自身的响应式状态，并调用 `storage.ts` 中的方法保存数据。
    3.  `stats.vue` 主动从 `storage.ts` 拉取数据，进行计算和展示。
*   **错误与日志**：在 `storage.ts` 和关键业务函数中加入轻量级日志记录（`console.log` 或写入日志文件），满足 **NF-003.2** 的可支持性要求。

#### **4. 外部接口与约束**

*   **用户接口**：由 Vue 3 实现的图形界面，通过 Tauri 打包为桌面应用窗口。
*   **软件接口**：Tauri 提供的系统API（用于文件系统访问、可能的消息通知）。
*   **通信接口**：无（离线应用，满足 **NF-002.1**）。
*   **约束**：
    *   必须为单机、离线桌面应用（**NF-005.1**）。
    *   界面响应与统计加载需满足性能要求（**NF-001.1**）。

#### **5. 部署与运行**

*   **开发**：使用 Vue 3 进行开发，Tauri 作为构建工具。
*   **打包**：通过 Tauri 将应用及其依赖打包为单一可执行文件（如 `.exe`, `.dmg`）。
*   **数据存储**：应用首次运行时，在用户标准应用数据目录（如 `%APPDATA%`、`~/Library/Application Support`）下创建专属文件夹，用于存放唯一的 `data.json` 文件和日志文件。

**总结**：本修正草案将架构简化为五个核心文件，每个文件功能内聚，边界清晰。通过最小化的接口进行必要通信，完全摒弃了复杂的多层架构，更加贴合使用 Vue 3 开发轻量级、本地化桌面工具的实际场景，能够有效支撑所有已定义的功能性（FR）与非功能性需求（NF）。
